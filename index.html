So what are they?

Rules your code should conform to, and will be checked against

Four language specific standards

  • PHP
  • HTML
  • CSS
  • JS

Accessibility (WCAG) Standards just landed!

Developed for wordpress core, themes & plugins

Written by the community, for the community

Have existed in some form or other since 2009

What they are NOT

Not just for the hardcore

Not just for core contributors

Not a definition of right & wrong

Just the "WordPress way"

Not there to make you feel guilty

Not going to add more complexity to your code

10 Benefits of Switching to WPCS

Reduced Stress

No more worrying about how you should write code Less stress = more code

Readable Code

Consistent indentation, spacing, structure and naming conventions

Good for Teams

All code looks the same, regardless of who wrote it or when it was written

Future Proofing

Avoid deprecated functions and use new or alternative functions instead

Faster Debugging

Bugs are quicker to find & fix because the code you write is easier to read & understand

Avoid Pesky Bugs

Use Yoda conditions, you must. Less buggy conditional statements, you will have

Better Inline Docs

Standards encourage more consistent and detailed documentation of your code

More Secure Code

Consistent validatation, sanitization and escaping of both input & output data

Better Querying

Touching the database directly is discouraged in favour of the Query APIs

Easier to Maintain

Anyone can modify your code to fix or improve it right now or in the future.

Switching to WPCS for writing PHP

What you will need to "sniff" your PHP as you develop

PHP Code Sniffer

Install PHPCS on your machine using PEAR or Composer. Homebrew is an alternative for Mac OSX users.

Add the path to your phpcs command to your system PATH, transforming it into a global installation

WPCS from GitHub

Download the WPCS rules for PHPCS to your machine using Git, Pear, Composer or the old fashioned way

Place them in a suitable directory and make a note of the location. Mine are in: /Users/davetgreen/.wpcs

A Code Editor / IDE

Sublime
Atom
Netbeans
PHPStorm
Coda

Or any similar software that either has a PHP Code Sniffer plugin available or comes with built-in support.

EDITOR SUPPORT

Install and configure the relevant plugin for your code editor / IDE, or turn on and configure native support.

You will need to provide the path to the directory containing the WPCS rules you downloaded from GitHub.
