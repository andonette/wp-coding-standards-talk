10 Benefits of Switching to WPCS

Reduced Stress

No more inner debates about how you write your code: just stick to the standards.

Easier to Read

Due to good use of spacing, indentation and braces.

Consistent Naming

Naming conventions for variables, constants, functions, classes and files.

#4 - Benefit Title

Good for Teams

All of your code will look like it was written by a single person.

#6 - Benefit Title

Better Inline Documentation

Of a higher quality and far more comprehensive.

More Secure Code

Consistent validatation, sanitization and escaping.

Avoid Pesky Bugs, You Shall

Using "Yoda", rather than regular conditions means that if you accidentally omit an equal sign, a variable cannot be set to the value of what you are comparing it against.

if ( 'strong' == $force ) {
    $victorious = you_will( $be );
}

Removing an equal sign from the condition would result in a parse error instead of the condition evaluating to true!

#10 - Benefit Title

Making the Switch

Automatically check your PHP as you write it.

PHP Code Sniffer

Install PHPCS on your machine using PEAR or Composer. Alternatively, use Homebrew if you are an OSX user.

Add the path to your phpcs command to your system PATH, transforming it into a global installation.

WPCS from GitHub

Download the WPCS rules for PHPCS to your machine using Git, Pear, Composer or the old fashioned way.

Place them in suitable directory and make a note of the location. Mine are in: /Users/davetgreen/.wpcs

A Code Editor / IDE

Sublime
Atom
Netbeans
PHPStorm
Coda

Or any decent code editor / IDE that either has a plugin ecosystem or native support for PHP Code Sniffer.

EDITOR SUPPORT

Install and configure the relevant plugin for your code editor / IDE, or turn on and configure native support.

You will need to provide the path to the directory containing the WPCS rules you downloaded from GitHub.
